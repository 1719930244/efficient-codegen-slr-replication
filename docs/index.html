Overview

📡
Living Resource — This companion website and the associated repository are continuously maintained. We regularly update the paper list with newly published studies on efficient LLM-based code generation. Last content update: April 2026.

Large Language Models (LLMs) have become indispensable tools for code generation, yet their computational costs—spanning data preparation, model training, inference, and evaluation—remain a critical barrier to practical adoption. This systematic review analyzes 122 primary studies to provide a comprehensive map of efficiency techniques across the full lifecycle of LLM-based code generation.

Our review spans five research questions aligned with the model lifecycle: data preparation (RQ1, 10 studies), model training (RQ2, 29 studies), inference optimization (RQ3, ~62 studies), deployment (RQ4, ~24 studies), and evaluation (RQ5, 21 studies). We identify 24 fine-grained technique categories and distill 7 key findings that reveal both progress and critical gaps in the field.

Key Findings

Our systematic analysis distills 7 key findings across the LLM code generation lifecycle:

Finding 1: Quality Dominates Quantity

Three complementary data-level strategies improve training efficiency with a common insight: quality dominates quantity. Training on fewer than 10% of carefully selected examples can match full-corpus performance; combining quality filtering with data selection yields larger gains than either alone; and curriculum ordering from simple to complex accelerates convergence by up to 30%. The optimal selection criterion and complexity metric vary by task, underscoring the need for strategies tailored to the target setting.

Finding 2: Minimal Parameter Updates Suffice

Training-level efficiency spans two complementary strategies with compounding benefits. Pre-training cost can be reduced by up to 40% through tokenizer optimization, dynamic packing, and sparse architectures. Parameter-efficient fine-tuning with LoRA and QLoRA matches full fine-tuning accuracy while updating fewer than 1% of parameters and reducing memory by up to 70%. Combining PEFT with data selection yields multiplicative gains across both data volume and parameter count.

Finding 3: What to Distill Matters More Than How

Knowledge distillation enables 7B-parameter students to approximate teachers over 10× larger. The critical design choice is what to distill: targeting the student’s weaknesses, transferring reasoning chains, or compressing multi-agent behaviors each yields distinct advantages over uniform distillation. RL for code LLMs serves two objectives, generation efficiency and generated code efficiency, but existing work predominantly targets the latter, leaving RL for generation process efficiency as a promising direction.

Finding 4: Adaptive Resource Allocation Outperforms Uniform

Inference techniques that adapt computation to task difficulty consistently outperform uniform strategies. Speculative decoding achieves 2× speedup, early exit reduces latency by up to 50%, and confidence gating in multi-agent systems cuts cost by 45%. The common principle is matching resource expenditure to actual need rather than applying fixed computation to every request.

Finding 5: Code Structure Enables Unique Optimizations

Compression that leverages code structure consistently outperforms generic methods, achieving up to 4× prompt compression with less than 2% accuracy loss. Irrelevant context actively harms generation quality, making well-targeted pruning a rare technique that improves both efficiency and accuracy simultaneously. Syntax-aware representations reduce token counts by up to 30%, demonstrating that the structural regularity of code creates optimization opportunities unavailable in natural language.

Finding 6: Deployment Compounds with Inference but Rarely Evaluated Together

Deployment-level techniques yield substantial standalone gains: quantization reduces memory by up to 70% with degradation that varies across languages, and model routing achieves over 30% cost reduction by directing queries to appropriately sized models. These deployment optimizations can compound with per-request inference techniques, but few studies evaluate such compositions, representing a significant gap between research and production deployment.

Finding 7: Efficiency Evaluation Lags Behind Optimization

Efficiency evaluation lacks standardization: latency is reported by 54% of studies, memory by 19%, energy by only 8%. Generation process efficiency as a first-class metric remains absent from existing benchmarks. Three priorities emerge: standardized reporting protocols, benchmarks that jointly measure correctness and generation cost, and unified cost models for comparison across techniques.
